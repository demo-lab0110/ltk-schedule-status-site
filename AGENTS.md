# AGENTS.md

このプロジェクトでは「Codex × ChatGPT 併用開発運用ルール v2」を前提に開発する。

## 作業前に確認すること

- 実装前に `docs/design.md` と、必要に応じて `docs/analytics.md` を確認する。
- 開発運用ルールは `docs/development-rules.md` を正とする。
- 設計書と矛盾する変更、MVP範囲を超える変更、URL、Hashルーティング、GA4、外部API、認証、権限、個人情報に関わる変更は、軽微に見えても実装前に停止して報告する。
- 今回やること / やらないことを作業前に明確にする。
- 既存機能への影響範囲を確認する。
- テスト条件と受け入れ条件を確認する。
- セキュリティ面を確認する。

## 役割分担

- ChatGPTは設計、判断、整理、レビューを担当する。
- Codexは実装、検証、差分報告を担当する。
- 最終判断と責任は人間が持つ。
- Codex作業後は、ChatGPTレビューを行う前提で報告する。

## 実装時の注意

- 既存UI、既存タブ挙動、既存データ取得処理を不用意に変更しない。
- URL付きタブ化は `/#/clips` のようなHashルーティングを優先する。
- GitHub Pages等の静的配信で404を起こす変更を避ける。
- GA4イベントにユーザー個人情報を送らない。
- URLに秘密情報を含めない。
- `gtag` 未定義時にエラーにしない。
- Draft Predictorのadmin系URLは、将来的な公開範囲に注意する。

## 作業後の報告形式

作業後は次を報告する。

- 実装内容
- 変更ファイル
- 設計書との整合性
- 今回やったこと
- 今回やらなかったこと
- テスト結果
- 既存機能への影響
- セキュリティ確認
- 人間が確認すべきこと
- 次にやるとよいこと
