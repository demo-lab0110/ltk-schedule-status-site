# Codex × ChatGPT 併用開発運用ルール v2

この文書は、LTK Schedule & Status サイトの開発運用ルール本体である。

## 基本方針

- ChatGPTは設計、判断、整理、レビューを担当する。
- Codexは実装、検証、差分報告を担当する。
- 人間が最終判断と責任を持つ。
- Codexの作業後は、ChatGPTレビューを行う前提で報告する。

## 実装前チェック

実装前に次を確認する。

- `docs/design.md` の設計内容
- GA4や計測に関わる場合は `docs/analytics.md`
- 今回やること
- 今回やらないこと
- 受け入れ条件
- テスト条件
- セキュリティ確認項目
- 既存機能への影響範囲

設計書と矛盾する変更、MVP範囲を超える変更、保存方式、データ構造、外部API、サーバー、認証、課金、通知、権限、個人情報に関わる変更は、実装前に停止して報告する。

## 実装時のルール

- 今回やること / やらないことを明確にしたうえで作業する。
- 既存UI、既存タブ挙動、既存データ取得処理を不用意に変更しない。
- 受け入れ条件を満たす変更だけを行う。
- テスト条件を確認し、実行できない場合は理由を報告する。
- セキュリティ確認を行う。
- 既存機能への影響を確認する。

## セキュリティ確認

- URLに秘密情報を含めない。
- GA4イベントにユーザー個人情報を送らない。
- `gtag` 未定義時にエラーにしない。
- Draft Predictorのadmin系URLは、将来的な公開範囲に注意する。
- 外部API、認証、権限、個人情報に関わる変更は、実装前に人間へ確認する。

## Codex最終報告

Codexは作業後、次を報告する。

- 実装内容
- 変更ファイル
- 設計書との整合性
- 今回やったこと
- 今回やらなかったこと
- テスト結果
- 既存機能への影響
- セキュリティ確認
- 人間が確認すべきこと
- 次にやるとよいこと

## ChatGPTレビュー

Codexの最終報告後、必要に応じてChatGPTが設計、仕様、リスク、受け入れ条件の観点でレビューする。

ChatGPTレビュー後も、最終判断と責任は人間が持つ。
